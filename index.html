How does JS work?

JavaScript is a programming language that runs on the client-side and is one of the most efficient and widely used scripting languages. JavaScript and JavaScript engines are both available. Google Chrome, for example, has its own JavaScript engine called V8. When we execute a JavaScript application in a web browser, the browser's engine receives the JavaScript code and runs the source code to produce the result. Sample flow-chart javascript engine work:
Java script file -> parser -> AST (Abstract syntax tree) -> Conversion to Machine Code -> Convert Machine Code ->Code runs

The parser's role is to verify the JavaScript line of code for syntactic problems. When the parser detects an error, it throws an error and stopped the execution of the code. It prepares the AST data structure when there are no errors in the code (it stands for Abstract Syntax Tree). Then parser creates an Abstract Syntax Tree, and the JavaScript engine turns the JavaScript language into machine code. When a JavaScript program is transferred to machine language, the converted code is provided to the system for execution, and the engine then executes the byte code.

What is the difference between Local storage & Session storage?

This read-only interface element gives access to the Document's local storage object, which keeps your data safe across browser sessions. Local Storage data is cleared when the page session ends – that is, when the page is closed – unlike session Storage data.

    1. There is no data transfer to the server.
    2. It is supported by fewer older browsers.
Local storage has 4 methods:
  • setItem() Method
  • getItem() Method
  • removeItem() Method
  • clear() Method
    • The sessionStorage read-only property may be used to access Session Storage objects. SessionStorage differs from localStorage in that localStorage data does not expire, however sessionStorage data is removed after the page session finishes.
